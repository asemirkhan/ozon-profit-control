# Ozon API Access — правила доступа и безопасности (v1)

## 1) Что нужно от продавца Ozon
Для подключения по API нужны 2 значения:
- `Client-Id`
- `Api-Key`

Это стандартные ключи доступа к Ozon Seller API.

## 2) Как объяснить клиенту безопасность (простыми словами)
- Доступ нужен только для чтения данных (продажи, финансы, реклама — по согласованию).
- Ключи не публикуются и не попадают в git.
- Ключи хранятся в переменных окружения (или в секрет-хранилище на сервере).
- При необходимости клиент может в любой момент отозвать ключи в кабинете.

## 3) Где хранить ключи в проекте
Ключи НЕ хранятся в файлах репозитория.

Хранение:
- в `.env` (локально) — файл в `.gitignore`
- на сервере (VPS/managed) — переменные окружения или секреты провайдера

## 4) Какие env переменные использовать
Рекомендуемые переменные окружения:
- `OZON_CLIENT_ID`
- `OZON_API_KEY`

Для Postgres уже используются:
- `PGHOST`, `PGPORT`, `PGDATABASE`, `PGUSER`, `PGPASSWORD`

## 5) Минимальный сценарий подключения (первый запуск)
Первый тест делается через справочник товаров:
- запрос списка товаров (product list)
- сохранение raw JSON
- загрузка в `dim_product`

Это позволяет убедиться, что:
- ключи валидны
- доступ работает
- коннектор и raw store работают
- база наполняется

## 6) Второй шаг после product list
После успешного `dim_product` подключаются:
- postings (отправления)
- transactions (финансовые операции)
- ads (реклама, если включено)
